The primary query ("Chrome extension or TypeScript tool to export open tabs to JSON or Markdown, for integration with AI agents...") feels exactly right. It captures the core need—metadata-first (URL, title, favIconUrl), optional content fetch, CLI/Node-readable output, extensible patterns, and cross-platform (no AppleScript)—while aligning perfectly with pi-mono's philosophy of minimal, skill-based extensions for agents. The alternatives are narrower subsets and less ideal for seamless AI integration.

Quick Existing Solutions (Off-the-Shelf)

These handle (1) metadata extraction well via chrome.tabs.query() + chrome.tabs.Tab (which includes url, title, favIconUrl, pinned, etc.). None do full page content by default (that needs per-tab content scripts + "scripting" + host perms), but you can fork any for it. Output is clipboard/download (easy for agents to consume via file read or clipboard tool).

TabCopy (most popular/recommended for your use case):
Exports current window / all windows to JSON, Markdown, Title+URL, CSV, HTML, custom templates. Clipboard output (super fast for AI paste-in). Custom formats + shortcuts + filtering. Includes title/URL reliably; favicon not in standard templates but available in the tab object if you tweak.
→ Chrome Web Store: https://chromewebstore.google.com/detail/tab-copy/micdllihgoppmejpecmkilggmaagfdmb
→ GitHub (for forking): https://github.com/hansifer/tab-copy
Great for quick "dump → feed to pi agent".
TabsDump (open-source, metadata-focused, tiny):
Popup → one-click export of title, URL + pinned/active/audible/muted/incognito/windowId (full chrome.tabs.Tab subset). JSON or plain text, auto-copies to clipboard. Vanilla JS (easy to fork to TS, add favIconUrl, switch to file download via chrome.downloads).
→ GitHub: https://github.com/danilo-znamerovszkij/TabsDump
Perfect starter if you want to customize output for Node/CLI consumption.
Tabs to Markdown or Active Tab Exporter:
Direct .md download (list of [title](url)) or JSON/TXT/CSV file. Simple, no fuss. FavIcon not included.

None pipe directly to stdout (extensions are sandboxed), but download-to-Downloads + a pi skill that reads the latest file works fine.

Best Fit for pi-mono / Agent Frameworks: Extend the Existing browser-tools Skill (CDP-Based)

pi-mono (badlogic/pi-mono) has no built-in open-tabs exporter, but its ecosystem is designed for exactly this kind of extension.

The separate pi-skills/browser-tools already connects to a running Chrome via Chrome DevTools Protocol (CDP) on :9222.
Setup is one-time: npm install in the skill dir + start Chrome with their browser-start.js (fresh or copy your real profile with cookies/logins intact). Tools include navigate, JS eval, screenshot, extract readable Markdown via Readability/Turndown, cookies, element picker.
Outputs are plain text / Markdown / file paths / JSON (via browser-eval.js + JSON.stringify).
Why this wins for AI-agent integration:
Extensible pattern matches pi's skills/extensions (TypeScript/Node scripts loaded on-demand, no prompt-cache bloat).
Add one new script (e.g. browser-list-tabs.js): connect via CDP (chrome-remote-interface or puppeteer-core), call Target.getTargets() → get {targetId, title, url, type} for all open tabs, enrich with Page.getResourceTree or attached session for favIconUrl if needed, output clean JSON/Markdown to stdout or file.
Agent calls it like any other tool: list_open_tabs --json → pipe result straight into pi-coding-agent context or another skill.
Optional content fetch: reuse their browser-content.js logic or eval JS per tab.
Fully CLI/Node-native, no extension needed once Chrome is debug-enabled.
Cross-platform (Linux/macOS/Windows), avoids macOS-only tools.

This is the most "pi-native" and extensible route. Fork the skill repo, add the tool, drop it in ~/.pi/agent/skills/, and your agent can use it immediately. Similar patterns exist in other minimal agents (Claude Code's Chrome native-messaging integration, etc.).

const tabs = await chrome.tabs.query({});
const data = tabs.map(t => ({
  url: t.url,
  title: t.title,
  favIconUrl: t.favIconUrl,
  // ... pinned, active, etc.
}));
// Option A: download JSON/MD
const blob = new Blob([JSON.stringify(data, null, 2)], {type: 'application/json'});
chrome.downloads.download({url: URL.createObjectURL(blob), filename: 'open-tabs.json'});
// Option B: native messaging to your Node CLI for stdout pipe
Permissions: "tabs", optional "scripting" + "<all_urls>" for content fetch via chrome.scripting.executeScript({target: {tabId}, func: () => document.body.innerText}).
Extensible: expose as web-accessible resource or native host.
Headless/CLI-only via CDP (no extension):
npm i chrome-remote-interface or puppeteer-core. Requires Chrome --remote-debugging-port=9222 (same as pi's browser-tools).
TypeScript
Copy
const CDP = require('chrome-remote-interface');
CDP(async (client) => {
  const {Target} = client;
  const targets = await Target.getTargets();
  const tabs = targets.filter(t => t.type === 'page').map(t => ({title: t.title, url: t.url /* favIcon via Page.getFrameTree if attached */}));
  console.log(JSON.stringify(tabs)); // stdout!
});
Clean, Node-native, perfect for pi integration. Downside: needs the debug flag (or restart Chrome).
