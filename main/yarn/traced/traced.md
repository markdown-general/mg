# Closing the Loop: Free Traced Categories in Haskell

*A Functional Pearl*

---

## Abstract

We present `Traced`, a GADT over Haskell functions with three
constructors: `Apply`, `Compose`, and `Untrace`. We show that `Traced`
is the free traced monoidal category generated by Haskell functions;
that its fold/build adjunction guarantees fusion and erasure at compile
time; and that it provides a canonical substrate for stream processing,
coroutines, and lateral computation — all without the `arr` baggage of
`Arrow` or the interaction problems of monad transformer stacks.

The central observation is a distinction that the Haskell ecosystem has
consistently elided: `Costrong` and `Cochoice` are not two spellings of
the same idea. `Cochoice` — the profunctor class underlying `Pipe`,
`MachineT`, `Conduit`, and `Streamly` — models sequential handoff: one
token, one step, one commitment. `Costrong` — the profunctor class
underlying `Traced` — models simultaneous feedback: the loop and the
result, always together. Every streaming library is `Cochoice`.
`Traced` is `Costrong`, and can derive `Cochoice` for free.

The key insight is a slogan: *keep sliding until yanked*. `Apply`
defers function application. `Compose` defers composition. `Untrace`
defers the fixed point — it keeps the loop open, sliding it through
compositions until `yank` closes it. `run` is the normalization
procedure that implements the sliding and collapses all three
constructors to a plain Haskell function.

---

## 1. The Problem

Consider implementing `(≤)` on Church-encoded naturals:

```haskell
newtype N = N { nat :: forall a. (a -> a) -> a -> a }

(≤) :: N -> N -> Bool
n ≤ m = nat n ns nz (nat m ms mz)
  where
    ns nk mk = mk nk   -- hand control to m's fold
    nz mk    = True
    ms mk nk = nk mk
    mz nk    = False
```

GHC rejects this:

```
Could not construct infinite type: t ~ (t -> Bool) -> Bool
```

The program is correct in an untyped setting. The two folds pass
control back and forth — they are coroutines running in lock-step. But
Haskell's type system cannot unify `t` with `(t -> Bool) -> Bool`.
Something is missing.

What is missing is not a type class, a newtype, or a language
extension. What is missing is a primitive notion of *feedback* — a way
to hold a loop open between two computations without collapsing it
immediately into a value. The type the typechecker rejects is not
wrong. It is the type of a feedback variable, and Haskell has no place
to put it.

This paper provides that place.

## 2. The Construction

We introduce `Traced`, a GADT with three constructors:

```haskell
{-# LANGUAGE GADTs, RankNTypes, ScopedTypeVariables #-}

module Traced where

import Control.Category
import Data.Profunctor
import Data.Profunctor.Strong (Costrong(..))
import Control.Monad.Fix (fix)
import Prelude hiding ((.), id)

data Traced a b where
  Apply   :: (b -> c) -> Traced a b -> Traced a c
  Compose :: Traced b c -> Traced a b -> Traced a c
  Untrace :: Traced (a, c) (b, c) -> Traced a b
```

Each constructor defers exactly one operation. `Apply` defers function
application to the output — a `fmap` that has not happened yet.
`Compose` defers sequential composition — a pipeline that has not been
connected yet. `Untrace` defers the fixed point — a loop that has not
been closed yet. The `c` in `Untrace` is the feedback variable: it
travels alongside the morphism, existentially quantified, invisible
from outside, waiting.

The two fundamental operations are `build` and `run`:

```haskell
-- Inject a Haskell function into Traced
build :: (a -> b) -> Traced a b
build f = Apply f (build id)

-- Collapse a Traced morphism to a Haskell function
run :: Traced a b -> (a -> b)
run (Apply f p)   = f . run p
run (Compose g h) = case g of
  Apply f p     -> f . run (Compose p h)
  Compose g1 g2 -> run (Compose g1 (Compose g2 h))
  Untrace p     -> \a -> fst $ fix $ \(b, c) -> run p (run h a, c)
run (Untrace p)   = \a -> fst $ fix $ \(b, c) -> run p (a, c)
```

The `Compose` case deserves attention. `run` does not recurse blindly —
it *inspects* the left argument before deciding what to do. This
Mendler-style case analysis serves two purposes: it reassociates
left-nested `Compose` chains, implementing associativity definitionally;
and it detects when `Untrace` appears on the left of a `Compose`,
triggering the sliding law. This is not a trick. It is the operational
content of the traced monoidal axioms, compiled into a normalizer.

The central law is:

```haskell
run . build = id   -- the fusion law
```

This holds coinductively: `build f` produces an infinite chain of
`Apply id` constructors, and `run` collapses them all to identity,
leaving `f`. Every static `Traced` pipeline visible to GHC at `-O2`
is erased entirely — the constructors disappear and only the underlying
function composition remains. This is fold/build fusion applied to the
traced category structure.

Running a closed loop is:

```haskell
yank :: Traced a a -> a
yank = fix . run
```

Collapse the morphism to a function, then take its fixed point. The
yanking axiom follows immediately:

```haskell
yank (build id) = fix . run . build $ id = fix id = id
```

## 3. The Instances

```haskell
instance Category Traced where
  id  = build id
  (.) = Compose

instance Functor (Traced a) where
  fmap f = Compose (build f)

instance Profunctor Traced where
  dimap f g p = build g `Compose` p `Compose` build f

instance Costrong Traced where
  unfirst = Untrace
```

Four instances. All one-liners. The `Costrong` instance is the entire
point: `unfirst = Untrace` says that the categorical trace operation
*is* the `Untrace` constructor. Feedback is not derived. It is
primitive. It is a constructor.

---

## 4. The Laws

### 4.1 Adjunction

```
run . build = id                          [fusion]
run (build (run p)) = run p               [idempotency]
```

Fusion is the fold/build rule. Idempotency follows immediately:
`run (build (run p)) = run p` by fusion with `f = run p`.

### 4.2 Category

```
run (Compose (build id) p) = run p        [left identity]
run (Compose p (build id)) = run p        [right identity]
run (Compose (Compose f g) h)
  = run (Compose f (Compose g h))         [associativity]
```

Associativity is definitional in `run` — the Mendler inspection
reassociates before recursing. `run` is not a catamorphism. It is a
normalizer, and associativity is part of what it normalizes.

### 4.3 Costrong

```
unfirst (dimap (id *** f) (id *** f) p) = unfirst p   [dinaturality]
unfirst . unfirst = unfirst . dimap assoc unassoc      [coherence]
```

Dinaturality holds by parametricity. The `c` in
`Untrace :: Traced (a, c) (b, c) -> Traced a b` is existentially
quantified. Once `Untrace` is applied, `c` is invisible from outside.
Any `f :: c -> c` acting on the feedback variable is unobservable, and
by parametricity must behave as `id`. This is the same argument that
makes `runST` safe: the `s` in `ST s a` cannot leak because it is
existential. The feedback variable in `Untrace` cannot leak for the
same reason.

### 4.4 The Sliding Law

```
trace(f) ∘ g = trace(f ∘ (g × id_c))
```

In `Traced`: when `Untrace` appears on the left of `Compose`, it
slides inside — absorbing `h` into the non-feedback component — rather
than blocking:

```haskell
run (Compose (Untrace p) h)
  = \a -> fst $ fix $ \(b, c) -> run p (run h a, c)
  = run (Untrace (Compose p (build (\(x, d) -> (run h x, d)))))
```

This is the operational heart of "keep sliding until yanked". Every
`Untrace` keeps sliding left through `Compose` chains — absorbing
whatever is to its right into the feedback rewiring — until there is
nothing left to slide past. At that point, `yank` closes the loop.
The loop was never prematurely closed. It waited.

## 5. The Tower

`Traced` is the top of a three-level tower of free constructions, each
adding exactly one deferred operation:

```
Level 1: Coyoneda  — free functor         — Apply   — defers fmap
Level 2: Free      — free category        — Compose — defers composition
Level 3: Traced    — free traced category — Untrace — defers the fixed point
```

At each level the adjunction is `build ⊣ run` and the fusion law is
`run . build = id`. The laws accumulate: `Traced` satisfies all the
laws of a free functor, all the laws of a free category, and adds the
traced monoidal laws on top.

We could present the three levels as three separate types. We flatten
them into a single GADT because the tower is conceptual scaffolding —
the reader does not need to see `Coyoneda` and `Free` as live code to
understand the progression. What matters is that each constructor
corresponds to exactly one level, `run` collapses all three, and the
structure costs nothing at compile time when it is not needed.

## 6. Keep Sliding Until Yanked

Monads close the loop at bind. The moment you write `m >>= f`, the
feedback is resolved: `f` receives a value, the loop is gone, the
computation moves on. You cannot inspect the loop. You cannot delay it.
You cannot pass it to a scheduler, an optimizer, or another
computation. `>>=` is `Untrace` applied immediately and irrevocably at
every step.

This is fine when you want sequencing. It is a problem when you want
feedback as a *value* — when the loop itself is the thing you need to
manipulate.

```haskell
-- Monad: loop closed at bind, c disappears
do x <- ma        -- Untrace applied here, right now
   f x            -- x is a value; the loop is already gone

-- Traced: loop deferred, c still in scope
Compose (Apply f (build id)) ma   -- loop is open
                                  -- c is a constructor argument
                                  -- run closes it when you say so
```

The monad transformer stack is trying to compose closed loops. When
`StateT` and `ExceptT` interact, you are asking two loops that were
already closed — at their respective bind sites — to cooperate after
the fact. They cannot, and the interaction laws fail. `lift` is the
sound of someone trying to reopen a loop that was sealed at bind time.

`MonadFix` is the Haskell community noticing this problem and bolting
`mfix` onto monads as a retrofit. But `mfix` is `Untrace` inside the
monad, which means the feedback variable is still trapped inside `m`.
You can loop, but you cannot hand the loop to anything outside `m`.

`ArrowLoop` gets closer. Its `loop :: a (b, d) (c, d) -> a b c` has
exactly the type of `Untrace`. But `Arrow` requires `arr :: (b -> c) ->
a b c`, which forces everything through `(->)` and makes the free
construction impossible. `ArrowLoop` is where the traced monoidal
category got stuck in Haskell: the right type, the wrong typeclass
hierarchy.

`Traced` is what you get when you take `ArrowLoop`, remove `arr`, and
keep only `loop`. The freedom comes entirely from refusing to close the
loop early. Keep sliding until yanked.

### The Recursion Schemes Angle

`Traced` is the *initial algebra* for the functor that generates
`Apply`, `Compose`, and `Untrace`. This is the recursion schemes
framing: the functor is the shape, the carrier is the content, and
`run` is the catamorphism.

Different encodings of stream processing are different carriers for the
same algebra. Moore machines, hyperfunctions, continuation-passing
streams, Mealy machines — they all receive a unique map from `Traced`
by the universal property of the initial algebra. You pick your
encoding by picking your carrier. The shape — three constructors —
stays the same.

Kmett's `folds` library is one carrier. `Hyp` is another. `MachineT`
is another. They are all, in this sense, the same thing: a `Traced`
catamorphism with a different target. The `machines` library is
particularly close — its `Process` category, `fit`, and `~>` connectors
are `Traced` morphisms and `Compose` in narrow disguise. What `machines`
lacks is `Untrace`. There is no `loop` in `machines`. The feedback
variable has nowhere to live. This is not an oversight — it is the
consequence of a design choice that we will now name precisely.

## 7. Costrong, We Get Along; Cochoice, Each a Voice

The Haskell profunctor hierarchy contains two classes for eliminating a
variable from a morphism:

```haskell
class Profunctor p => Costrong p where
  unfirst  :: p (a, d) (b, d) -> p a b   -- product tensor
  unsecond :: p (d, a) (d, b) -> p a b

class Profunctor p => Cochoice p where
  unleft  :: p (Either a d) (Either b d) -> p a b   -- coproduct tensor
  unright :: p (Either d a) (Either d b) -> p a b
```

Same shape. Different tensor. The difference is everything.

`Cochoice` / `Either`: at each step the machine is *either* doing work
*or* looping. Never both simultaneously. One token moves through the
system at a time. This is sequential handoff — each participant gets a
turn, hands off to the next, waits. `MachineT`, `Pipe`, `Conduit`,
`Streamly` — all `Cochoice`. They yield *or* await *or* loop. The
`Either` is the branch.

`Costrong` / `(,)`: the feedback variable travels *alongside* the
output. Both exist simultaneously. The loop and the result are produced
together, in the same step, always. This is genuine simultaneity — two
things happening at once, coordinated through the shared `c`.
Hyperfunctions, `ArrowLoop`, `MonadFix` — all `Costrong`.

In one sentence: `Cochoice` is taking turns. `Costrong` is holding hands.

`Traced` is `Costrong`. The streaming libraries are `Cochoice`. This is
not a design preference. It is a mathematical fact about which tensor
you choose.

The practical consequence is visible whenever a `Cochoice` library
tries to do something that requires simultaneity: `merge`, `zipWith`,
concurrent pipelines, self-referential processes. Each one requires a
special combinator. Each one breaks the composition laws slightly. Each
one is secretly approximating `Untrace`. The library is trying to
simulate `(,)` with `Either`, and it never quite closes.

`Traced` has one combinator: `Untrace`. Everything else is derived.

```haskell
-- Cochoice derived from Costrong: sequential handoff from simultaneous feedback
instance Cochoice Traced where
  unleft p = build $ \a ->
    let go (Left b)  = b
        go (Right d) = go (run p (Right d))
    in go (run p (Left a))
```

The `Either` loop is implemented by closing a `(,)` loop and
projecting. `Cochoice` is `Costrong` with extra steps.

The `machines` library makes this concrete. Its `Step` functor:

```haskell
data Step k o r = Stop | Yield o r | forall t. Await (t -> r) (k t) r
```

is a coalgebra for `Cochoice` — `Yield` and `Await` are the two
branches of the `Either`, and `Stop` is the terminus. A `MachineT` is
the greatest fixed point of `Step`, which is precisely the final
coalgebra to `Traced`'s initial algebra. `Traced` generates;
`MachineT` observes. The catamorphism from `Traced` to `MachineT` is
the compilation step — build your pipeline in `Traced`, keep your
`Untrace` constructors sliding, then compile to `MachineT` at the
boundary for execution. The missing operation in `machines` is `loop`,
and it is missing because `Step` has no room for it. Adding `loop` to
`machines` would mean adding a `Costrong` constructor to `Step` — which
would make it `Traced`.

## 8. The Hyperfunctions Connection

Kidney and Wu define hyperfunctions as the greatest fixed point:

```haskell
newtype a ↬ b = Hyp { ι :: (b ↬ a) -> b }
```

with combinators:

```haskell
(⊳) :: (a -> b) -> (a ↬ b) -> (a ↬ b)   -- Apply
(⊙) :: (b ↬ c) -> (a ↬ b) -> (a ↬ c)   -- Compose
run :: a ↬ a -> a                          -- yank
```

`Traced` is the initial algebra to `Hyp`'s final coalgebra. The
correspondence is exact:

| `Traced`    | `Hyp`    |
|-------------|----------|
| `Apply f p` | `f ⊳ h`  |
| `Compose`   | `⊙`      |
| `Untrace`   | `ι`      |
| `yank`      | `run`    |
| `build f`   | `rep f`  |

The bridge:

```haskell
toHyp :: Traced a b -> a ↬ b
toHyp (Apply f p)   = f `push` toHyp p
toHyp (Compose g h) = toHyp g `compose` toHyp h
toHyp (Untrace p)   = Hyp $ \k -> fst $ fix $ \(b, c) ->
                        ι (toHyp p) (Hyp $ \_ -> (run k, c))

fromHyp :: a ↬ b -> Traced a b
fromHyp h = build (run . (h `compose`) . build)
```

`toHyp . fromHyp = id` by the universal property of `Hyp`. `fromHyp .
toHyp ≠ id` intensionally — `Traced` carries more structure than `Hyp`
exposes through `ι`.

The practical difference is transparency. `Hyp` is opaque: you can
only observe it through `ι`, one step at a time. `Traced` is
transparent: you can pattern match on its constructors, inspect its
structure, and let GHC fuse through it. `Traced` is the version you
want for a library. `Hyp` is the version you want for a proof.

Kmett's `hyperfunction` package presents `Hyp` directly, with `push`,
`compose`, and `run` as the interface. It is correct but opaque. It
does not name `Costrong`. It does not connect to the fold/build
adjunction. It does not observe that `Hyp` is a `Traced` catamorphism
target. The initial algebra is missing from the picture entirely, and
without it the relationship to streaming, fusion, and the profunctor
hierarchy is invisible.

## 9. The Examples

We translate the key examples from Kidney and Wu into `Traced`.

### 9.1 Church-Encoded Naturals: `(≤)`

The infinite type from Section 1 resolves immediately. The feedback
variable that GHC could not unify is the `c` in `Untrace`, held open
until `yank` closes it:

```haskell
(≤) :: N -> N -> Bool
n ≤ m = yank $ Compose
  (nat n (Apply id) (build (const True)))
  (nat m (Apply id) (build (const False)))
```

The two folds are composed and yanked. No infinite type. The fixed
point that GHC rejected as a type is now the operational semantics of
`yank`.

### 9.2 Subtraction on Church Naturals

```haskell
sub :: N -> N -> N
sub n m = N $ \s z -> yank $ Compose
  (nat n (Apply id) (build (const z)))
  (nat m (Apply id) (build s))
```

`n` contributes `id`s followed by `const z`. `m` contributes `id`s
followed by `s`. Composed and yanked, the result is `n - m`
applications of `s` to `z`.

### 9.3 Zip on Church-Encoded Lists

```haskell
zip :: [a] -> [b] -> [(a, b)]
zip xs ys = yank $ Compose
  (foldr (\x k -> Apply (prod x) k) (build (const [])) xs)
  (foldr (\y k -> Apply (cons y) k) (build (const [])) ys)
  where
    prod x rest = \consume -> consume x rest
    cons y emit = \x rest -> (x, y) : rest emit
```

The producer fold builds a `Traced` pipeline emitting `x` values. The
consumer fold builds a `Traced` pipeline consuming them, pairing with
`y`. `Compose` connects them. `yank` closes the loop.

### 9.4 The Concurrency Monad

Claessen's concurrency monad, with `Traced` as the substrate:

```haskell
type Conc r m = Cont (Traced (m r) (m r))

atomC :: Functor m => m a -> Conc r m a
atomC ma = Cont $ \k -> Apply id (k =<< ma)

forkC :: Conc r m a -> Conc r m ()
forkC m = Cont $ \k -> Compose (runCont m (const (build id))) (k ())

runC :: Monad m => Conc r m a -> m r
runC c = yank (runCont c (const (build id)))
```

`Compose` is the scheduler. `yank` runs it. The concurrency monad is
`Traced` with a monad along for effects.

### 9.5 Pipes

Spivey's pipe implementation, revealed:

```haskell
type Producer o r = Traced (o -> r) r
type Consumer i r = Traced r (i -> r)

merge :: Consumer o r -> Producer o r -> Traced r r
merge = Compose

runPipe :: Traced r r -> r
runPipe = yank
```

`merge` is `Compose`. Running is `yank`. The pipe library is two type
synonyms and two one-liners, and all the composition laws are inherited
for free.

## 10. Related Work

**Joyal, Street, and Verity** introduced traced monoidal categories in
1996. The axioms — sliding, yanking, superposition, dinaturality — are
the categorical foundation of `Untrace` and `run`. The sliding law is
what `run`'s Mendler inspection implements. The yanking axiom is
`yank (build id) = id`.

**Launchbury, Krstic, and Sauerwein** introduced hyperfunctions. Their
`⊳`, `⊙`, and `run` are `Apply`, `Compose`, and `yank` in `Traced`.

**Kidney and Wu** (2026) catalogued hyperfunctions across the Haskell
literature — coroutines, CCS, concurrency monads, breadth-first
traversal — and connected them via a common interface. Their paper is
the direct inspiration for the examples in Section 9. What they show is
that the hyperfunction pattern recurs everywhere. What we add is the
name for why: all of these are `Traced` catamorphisms.

**Gill, Launchbury, and Peyton Jones** introduced fold/build fusion in
1993. The `run`/`build` adjunction is this rule applied to the traced
category structure. GHC's list fusion and `Traced`'s erasure are the
same mechanism at different levels of the tower.

**Kmett's `machines` library** is the closest the Haskell ecosystem has
come to `Traced` without naming it. `Process` is a `Traced` morphism.
`~>` is `Compose`. `fit` is `dimap`. What is missing is `loop` — and
its absence is the exact marker of the `Cochoice`/`Costrong` boundary.
`machines` is `Traced` with `Untrace` deleted.

**Kmett's `folds` and `hyperfunction` libraries** approach `Traced`
from the coalgebra side. The `Fold` type is a represented Moore machine
and, as Kmett observes, is isomorphic to `Hyp`. But the initial algebra
is absent. The fold/build adjunction is not named. The `Costrong`
instance is not given. The sliding law is not proved. The libraries are
correct but incomplete: they have the final coalgebra without the
initial algebra that universally generates it.

**`ArrowLoop`** has `loop :: a (b, d) (c, d) -> a b c` — exactly the
type of `Untrace`. This is where the traced monoidal category got stuck
in Haskell. The `Arrow` class requires `arr`, which forces everything
through `(->)` and makes the free construction impossible. `Traced` is
`ArrowLoop` minus `arr`.

## 11. Conclusion

`Traced` is three constructors and two functions:

```haskell
data Traced a b where
  Apply   :: (b -> c) -> Traced a b -> Traced a c
  Compose :: Traced b c -> Traced a b -> Traced a c
  Untrace :: Traced (a, c) (b, c) -> Traced a b

build :: (a -> b) -> Traced a b
run   :: Traced a b -> (a -> b)
yank  :: Traced a a -> a
yank  = fix . run
```

`Apply` defers application. `Compose` defers composition. `Untrace`
defers the fixed point. `run` slides and collapses. `yank` closes.

The instances — `Category`, `Functor`, `Profunctor`, `Costrong` — are
one-liners. The laws — fusion, identity, associativity, sliding,
yanking, dinaturality — all hold, with associativity and sliding
following from Mendler inspection in `run`, and dinaturality following
from parametricity of the existential `c`.

`Traced` is the initial algebra. Every streaming library, every
concurrency monad, every pipe, every Moore machine, every hyperfunction
is a carrier for the `Traced` catamorphism — a different way of
answering the same question: what does it mean to close a loop?

`Costrong`, we get along. `Cochoice`, each a voice.

The streaming libraries chose `Cochoice` and have been paying for it
ever since: special combinators for merge, workarounds for concurrent
pipelines, interaction laws that fail under composition. They are
simulating `(,)` with `Either`, one special case at a time.

`Traced` is `Costrong`. It holds the loop open — sliding it through
compositions, preserving its structure, keeping the feedback variable
alive — until the very end. Then `yank` closes it.

Keep sliding until yanked.
